Filtering: Interface & Layout

Truncate Filters with 10+ Values #470

Issue: Long lists of filter options push other available filter types out of view, while dedicated inline scrollable areas are difficult for users to control.

Advice: Truncate filter options if there are more than 10, and make sure that the “View More” link is easy to discern. Avoid both inline scrollable areas and untruncated lists with 10+ filter options.

Issue first observed Feb 2015. Guideline updated 6 times since, most recently Jan 2023.

Importance:Essential

Platform:Desktop Web

Distinctions:Missed Opportunity

Research Findings

When there are filter types that have many filter options, there are 3 common ways to present the options on desktop:​‌‌‌‍‌‌‌‌‌‌‍‌‌‌‍‍‌‌‌‌‌‌‎

  1. list all filter options,
  2. use a dedicated inline scrollable area, or
  3. truncate the list of filter options.

During desktop testing, participants tended to have difficulties with long lists of filter options regardless of how they were presented; however, it was observed that truncation of filter options performed best when implemented carefully.

In this guideline, we’ll discuss test observations for each of the 3 ways of displaying filter options on desktop sites (see #1126: Enable Mobile Users to Search Long Lists of Filter Options, and Order Options Alphabetically for how to design long lists of filter options on mobile).

1) List All Filter Options

When all the filter options for each filter type are displayed in one long list, as seen here on Pottery Barn, users cannot get an overview of the available filter types (a 1200x800px viewport is marked in red). During testing, this often led participants to overlook (or never scroll far enough to see) the filter types most relevant to them, which severely limited their product-finding abilities.

“Looking over here at the filters, it’s nice to see I’ve got a wide range, but it is quite overwhelming, and I’ve not even heard of a lot of these brands, “Urbanista”, “Volkano”, “Groove”.” Long filter option lists also caused issues during large-scale European testing, such as here on Currys (UK). Very long lists can overwhelm users.

1 additional example

“All right, okay. So I’m not liking this website”. At eVitamins, a participant was overwhelmed by the number of filter options in the untruncated list (first image), which takes multiple viewports to scroll to the bottom of before the other filter options are visible (second image).

We’ve identified another 15 examples of ‘lising all available filter options,’ illustrating this implementation type across 15 case studies and 7 different industries.

View 15 Desktop Web examples

When all available filter options are displayed in one long list, any filter type with a lot of options will push the other filter types down the page. During testing, participants often missed out on relevant filter types that were pushed out of the first viewport because they didn’t see them or, in some instances, selected less-suitable options that appeared higher in the list instead.

Furthermore, when filter types extend across multiple viewports, users have much more difficulty establishing an overview of which filter types are available to them — which in turn increases the likelihood that they’ll miss out on potentially ideal filters that are pushed too far down the page.

Long lists of filter options, therefore, can cause users to overlook suitable filter options, thereby significantly slowing the product-finding process.

2) Dedicated Inline Scrollable Areas

Another approach to displaying long lists of filter options is to limit the space they take up by displaying them in an inline scrollable list. While this solves the issue of not being able to establish an overview of all the available filter types, the use of inline scrollable areas proved troublesome during testing.

The following 5 issues with inline scrollable areas were observed during testing:

1) No Overview of Filter Options

Since only a handful of filter options are visible by default in various filter types on Sephora, users will find it difficult to get an overview of them, and to select between options that are not close together.

We’ve identified another 16 examples of ‘inline scrollable area for filters,’ illustrating this implementation type across 15 case studies and 6 different industries.

View 16 Desktop Web examples

While inline scrollable areas allow users to get an overview of the filter types, it simply shifts the issue to the filter options. With inline scrollable areas for filter options, it’s impossible to get an overview of all options because only a small part of the list is viewable at any given time. This also makes it difficult to compare filter options that are not positioned closely together.

2) Scroll Hijacking

Inline scroll areas in filter sidebars, such as here on Build.com, can hijack scrolling of pages. Notice how, when scrolling the page, if the cursor moves over the inline scrolling area the filter options are scrolled instead of the page. Scroll hijacking will annoy many users, and multiple occurrences will quickly escalate an annoying issue to a disruptive one.

When scrolling the page, inline scroll areas can accidentally hijack the page-scrolling feature. For instance, when scrolling the page using a mouse wheel, a user’s cursor may drift over an inline scrollable area, causing the inline area to be scrolled instead of the page as intended.

Similarly, when a user is deliberately scrolling an inline area, once the end of the area is reached the page will be scrolled again. This jerky scrolling experience annoyed test participants, who often expressed frustration with the lack of page control, requiring them to be constantly aware of the cursor position while scrolling.

3) Invisible Content & Scrollbars

On some sites, scrollbars are hidden by default, as seen here on eBags. This makes it difficult for users to realize that inline scrollable areas are used, making it likely that suitable filter options would be overlooked.

Sometimes, scrollbars in filter types are hidden when they are not in use. This makes it difficult for some users to realize that there are inline scrollable areas in the filter interface, and so may assume that the only filter options available are those that are visible by default. Overlooking filter options can have critical consequences — and was indeed a direct cause for site abandonments during testing.

4) Overly Sensitive Scrollbars

Long lists of filter options combined with a low scroll area height can make the scrollbar very sensitive. During testing, participants had serious issues controlling such highly sensitive scrollbars and often unintentionally scrolled past relevant filter options.

Some users will attempt to drag the inline scrollbar control instead of using, for example, a scroll wheel on a mouse. Although some do this out of habit or preference, many test participants who experienced scroll hijacking tried to drag the scrollbar control as a fallback strategy. When there is a long list of filter items and a limited scroll area height, scrolling the list can be difficult to control. Several participants jumped past many filter options without noticing simply because the scrollbar proved too sensitive.

5) Alternative Interactions

Some participants had issues interacting with inline scroll areas. Here a participant wanted the brands with “L” (Lego), so she first tried hitting the keyboard letter L and then tried using the up and down arrows, as she found the small embedded scrollbar very unstable.

Some test participants expected alternative interaction methods to work when the filter options were inside an inline scroll area. For example, some tried clicking within the inline scroll area to activate it, only to find that they accidentally applied a random filter option. Others expected the inline scrollable list to accept keyboard shortcuts, just as native drop-downs do, pressing either a keyboard letter (expecting to be taken to that part of the list) or using the up and down arrows (a relatively logical conclusion as a long drop-down in many ways is also an inline scrollable list). If alternative interaction methods don’t work with inline scrollable areas, finding and selecting suitable filter options becomes more troublesome than it should be.

Therefore, given the number and severity of issues associated with inline scrollable areas, they should be avoided.

3) Truncated Lists

Here on Best Buy (first image) and Newegg (second image), test participants scrolled down to look over the available filters types. The truncated filter options lists made it easy for them to get a complete overview of all the available filter types.

We’ve identified another 47 examples of ‘truncated filter lists,’ illustrating this implementation type across 47 case studies and 11 different industries.

View 47 Desktop Web examples

When designed carefully, truncation of the filter options proved to perform better for displaying more than 10 filter options than either displaying all filter options or using inline scrollable areas.

A truncated list allows users to get an overview of all the filter types available, but when expanded it also allows users to get an overview of the filter options within. Furthermore, users avoid any scrolling issues associated with inline scroll areas.

However, if designed poorly, truncation can be troublesome because it may mislead users into thinking that the truncated list represents all of the available options. Therefore, the primary design challenge to address with truncation is to make sure that users are immediately aware that the list of filter options is truncated and that additional options exist.

Implementation Details for Truncating Filter Options

During testing, the following 6 implementation details for truncation design were observed to have a profound impact on participants’ ability to successfully work with long lists of filter options:

1) Display up to 10 Options Before Truncating

REI displays 7 filter options before truncation. As a result, users will quickly realize that there are more options available, but the total list of options doesn’t push subsequent filter types too far down the page.

Displaying 6 to 10 filter options before truncating the rest of the list was observed to perform well during testing. If fewer options are displayed, some users may assume that there is no truncation, and that the full list is shown. The maximum number of options to display before truncation can vary, but beginning truncation somewhere before or at the 10th filter option is a good rule of thumb. Furthermore, to inform the user how many additional filters they can expect, consider including the number of hidden filter options in the truncation link (e.g., “See 18 more”).

2) Use an Appropriate Link Style

The underlined link style helps users at Crutchfield to distinguish the truncation link from filter options, which use a checkbox design instead. This helps users understand that it’s a clickable option.

The link style of the truncation link is vital in making the user notice that it’s a clickable option and that it fulfills a different function from the rest of the filter options. Ideally, the default sitewide link styling should be used for the truncation link to indicate that it’s clickable. However, because it isn’t a filter option to be applied, the truncation link obviously shouldn’t have a checkbox like the filter options should (see guideline #479). The lack of a checkbox helps to provide contrast between the two types of links and further indicates that they invoke different functionality.

3) Ensure Good Proximity

Toys’R’Us provides an example of good proximity, with the “More…” truncation link placed right below the filter options. The white space added between the filter types further clarifies which collection of filter options the “More…” links belong to.

The truncation link should be positioned close to the filter type it belongs to so users don’t mistake it as either a separate link or a link that belongs to any subsequent filter type. A change in the amount of indentation can also help differentiate the truncation link from the filter options.

4) Include a Visual Indicator

Truncation links on Etsy include a plus sign (“+ Show more”). This, together with the bold text, makes the links stand out to users and they are therefore unlikely to be overlooked.

Using visual indicators alongside the text helps to distinguish truncation links from filter options. The most commonly used visual indicators are plus signs or downward arrow icons.

5) Consider Fading the Last Filter Option

A simple mock-up of TigerDirect, showing how fading the last filter option (“Networking”). can make it more obvious that some of the filter options are hidden.

Gradually fading out the last filter option can help emphasize that the list extends beyond the visible options — the user can literally see that the list is incomplete because the last option is only partially visible. Furthermore, the fade can help draw the user’s attention toward the truncation link.

6) Never Truncate a Single Option

Displaying the truncation link (“1 More…”) takes up as much space as displaying the actual filter option would, adding unnecessary complexity to the filter interface. The truncation threshold should instead be offset by an extra item so it only truncates 2 or more options.

It simply doesn’t make sense to truncate a single option since showing the truncation link will take up the same space as showing the filter option that’s hidden. It’s only when 2 or more options are truncated that the desired space efficiencies can be realized and thus truncating the list is justified. This can be achieved by simply offsetting the truncation logic by one, so that the truncation threshold is actually 2 items off the filter option size that invokes a truncation interface.

1 additional example

“And then I always look for ‘Vegan’, so I’m going to see if… ‘Diet’. ‘See More Diets’. ‘Vegan’.” At LuckyVitamin, a participant was easily able to locate and expand both the filter category (first image), and the truncated list of filter options (second image), before ultimately locating the “Vegan” filter (third image).

Ensure That Filter Options are Easily Found

68% of our benchmark sites truncate filter options when there are more than 10, up from 58% in 2015. This is an encouraging trend in the industry, but sites that don’t adhere to this guideline are making it harder than necessary for users to find suitable filter options.

While it isn’t necessary to implement all of these design patterns, the more of them that are used the more immediately clear it will be that the list of filter options is truncated.

Additionally, when truncating filter options, be sure to order the displayed filter options by combined sales and not by number of products matching each filter option — see guideline #471.

At Tesco, a participant wanted to filter by the brand “Olympus”, but it wasn’t included among the nontruncated options so he clicked the “More” link (first image). This opened a lightbox with sufficient space to display all 37 brand filter options all within one viewport (second image). Had this list been simply shown without truncation in the sidebar, it would have been more difficult for the user to get an overview of all the filter options.

Finally, on a few of the tested sites, the “+ More” link didn’t expand the list in the sidebar but instead opened the full filter option list in an overlay. This method performed well during testing, as the additional space allowed for all the filter options to be displayed within one viewport. However, introducing an overlay is not without risk, as not all users will expect a new window to open and may be slowed down by its appearance. It’s therefore recommended to only open filter options in an overlay when there are at least as many filter options truncated as there are displayed (e.g., 20+ in total).
