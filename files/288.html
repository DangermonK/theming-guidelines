Consider Visually Separating Sitewide Courtesy Navigation from Main Product Navigation #288

Issue: Courtesy navigation can clutter the main navigation and make it more difficult to begin browsing products.

Advice: Sitewide courtesy navigation should be visually separated from the main product navigation and grouped into thematic sections. In general, “Sign In”, “Create Account”, “Help”, “Contact / Customer Service”, “Store Locator” (for omnichannel sites), “Track an Order”, and very popular site-specific options should be provided as courtesy navigation options.

Issue first observed Aug 2014. Guideline updated 9 times since, most recently Feb 2023.

Importance:Essential – Impactful

Platforms:Desktop WebMobile WebMobile App

Distinctions:Missed Opportunity

Research Findings

While browsing for products is the main task for e-commerce users, being able to easily navigate to secondary nonproduct content is still important. Courtesy navigation items include links to customer support sections, membership programs, store finders (for multichannel retailers), shipping and returns information, account creation and sign in, and information about the company. However, during testing, courtesy navigation sometimes ended up distracting users from their main goal of finding suitable products — which added delays to the product-browsing process and led to a sense that the main navigation was overly cluttered and unnecessarily difficult to understand. During testing, several options for separating courtesy navigation from main product navigation were observed, for both desktop and mobile sites.​‌‌‌‍‌‌‌‌‌‌‍‌‌‌‍‍‌‌‌‌‌‌‎

Desktop Implementation Details

On desktop sites, achieving the right balance between having a sufficient amount of courtesy navigation options, without having an overly cluttered header, can be difficult, but during testing the following techniques resonated well with participants.

1) Providing Clear Visual Separation from the Main Product Navigation

Ann Taylor places the courtesy navigation options above the product navigation menu, making it unlikely users will stumble over the courtesy navigation when trying to access the main product navigation.

Having a clear visual separation between the main product navigation and the secondary courtesy navigation, by separating them completely, by keeping the courtesy items as simple text links, or by using different background colors if integrated in the same navigation bar helps users avoid inadvertently accessing one navigation instead of another. This also helps them to mentally keep the paths distinct (i.e., “This block is for the courtesy navigation, this block is for the main navigation”), and will also reduce the perception that the navigation appears overly cluttered.

2) Grouping Courtesy Navigation Links into Sections

Northern Tool separates courtesy navigation options related to getting help (the phone number for customer support and chat) from other courtesy options to the right.

Having the courtesy navigation grouped into separate, thematically related, sections — for example, “Sign In”, “Create Account”, and “Sign Out” can form one group of text links, “Help”, “Contact”, and “Chat” can be a second group, “Shipping and Returns”, “Store Locator”, and “Track Order”, can be a third, etc. — can help users quickly access the courtesy navigation option they’re looking for.

3) Using Drop-Down Menus

GoPro’s “Help” drop-down menu allows users to easily access nonproduct links but without distracting users that want to start looking for products.

1 additional example

On Gucci, text within the courtesy dropdown clearly indicates the features that can be found within after signing into the user’s account.

Drop-down menus can be used to collapse some of the courtesy options if there are more than a handful of thematically related options provided. However, remember to separate the courtesy drop-down menu from the main drop-down menu to avoid accidental hover activation, and consider an extra-long hover-activation delay (see #289).

Walmart uses icons to indicate the courtesy navigation options, and then shows the relevant text when hovering, an interesting (though untested) approach for sites wanting a more simplified appearance for the courtesy navigation.

Additionally, ensuring that the sitewide header isn’t otherwise cluttered with links, ads (see #240), or overly distracting images will help ensure users can find their way to both the main product navigation and the courtesy navigation options.

Mobile Implementation Details

Separating courtesy navigation options from the main product category paths doesn’t mean the courtesy options have to be placed last. For some sites — for example, mass merchants like Walmart — it may be appropriate to emphasize the courtesy navigation options in the main navigation. Note, however, that traditional product categories are still visible below the courtesy navigation options, and that the two types of navigation are visually separated with a line.

The courtesy navigation on L.L. Bean (first image), and Williams Sonoma (second image) are easily accessible, but don’t interfere with users that just want to start browsing immediately. Visual separation such as a strong green line on L.L. Bean and the placement of the courtesy navigation links in much smaller text above the site name on Williams Sonoma makes it clear to users that the two types of navigation have separate functions.

During mobile testing, many sites opted to place the courtesy navigation items in the main navigation menu, although others included a few options in the header or on the homepage as well. Because of the smaller viewport on mobile, it was often harder for test participants to ignore the courtesy navigation options. Thus, more participants were liable to be distracted by courtesy navigation during mobile testing than during desktop testing. When there’s no visual separation or distinction between the courtesy navigation options and the traditional product categories, many users will have a harder time finding the path they’re looking for in the main navigation — and, consequently, will find it harder to find the product they’re looking for.

On mobile, it’s therefore critical that the courtesy navigation options are visually separated from the product-category paths in the main navigation. Courtesy navigation can be separated from the main product navigation by making it visually distinct, separating using white space or a horizontal separator, using icons, or using a different background color. In short, it should be immediately clear to users when they open the main navigation that there are two “sections” of navigation — the main product navigation and the courtesy navigation — and it should be clear which links belong to which section through visual differentiation alone without having to actually read the link text.

Which Courtesy Options to Include

To better understand which courtesy options are the most heavily used by users, we conducted a quantitative study of 3,797 US adults. Results indicate that order tracking and account sign in should always be included on sites with these features available, followed by store locators, account creation, and contact options.

Finally, what courtesy options should actually be provided is somewhat determined by a specific site’s particular context. However, testing found that the following options should generally be included as courtesy navigation options:

  • “Sign In”
  • “Create Account”
  • “Help”
  • “Contact / Customer Service”
  • “Store Locator” (for omnichannel sites)
  • “Track an Order”
  • Very popular site-specific options (e.g., popular account features such as lists or rewards programs)

Providing these courtesy navigation options should give most users the paths they need when trying to access nonproduct-related content.

Guideline: Sitewide courtesy navigation should be visually separated from the main product navigation and grouped into thematic sections. In general, “Sign In”, “Create Account”, “Help”, “Contact / Customer Service”, “Store Locator” (for omnichannel sites), “Track an Order”, and very popular site-specific options should be provided as courtesy navigation options.
